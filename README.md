# KGC_project
영업 마케팅 차세대 1차 프로젝트 (22.9 ~ 23.6) - 약 70억
경쟁입찰 - 착수 - 분석 - 설계 - 개발 - 테스트 - 오픈

1. 경쟁입찰 (총 10개 수행사 지원 및 기술평가80 + 가격평가 20)
   7월 RFP 쓰고 8월동안 접수받아서 진행 완료
2. 착수 프로젝트 킥오프 23년 9월 16일자로 시작
3. 프로젝트 분석단계 진행 (수행사 투입 인력 총 43명(TA/AA/개발자/대응개발자/마이그레이션 등) 환경 세팅 및 보안 계정, 필요 권한/방화벽 신청)
4. 감리일정(1차, 2차, 3차)가 있는 프로젝트
5. 분석단계 9월 ~ 11월 / 설계 11월 ~ 1월 중순까지
6. 설계와 개발단계 병행으로 개발은 24년 2월 본격적으로 진행되면서, 24년 4월 12일까지 개발단계
7. 단위테스트 3월 말부터 진행해서 4월 둘째주까지 결항 등록 및 조치 시정하였습니다.
8. 통합테스트는 현업(디지털마케팅, 영업, CRM, CCM 등 유관부서) 분들과 1,2차로 진행 예정으로 계획을 세웠습니다. (관련 환경 세팅 진행)
   - 1차 통합테스트 기간 : 24년 4월 15일 ~ 26일
   - 2차 통합테스트 기간 : 24년 4월 29일 ~ 14일
   -> 1차 통합테스트 기간 중에 3차 감리 기간이 겹쳐서 수행사 분들께서 많이 고생하셨다.(이 시기 수행사 PM님 많이 힘들어보이셨던..)
9. 오픈은 5월 말일에서 6월 1일 넘어가는 00시에 레거시(POS, 정몰, 케어나우) 올 다운.. -> 협의가 완료된 부분 -> 6월 1일, 2일 주말동안 실제 운영처럼 반영해서
    테스트해보게끔 회사에서 지원하기로..(이때 매출은...)
   정식 6월 3일 오픈 ! 을 예정으로 하고있으며, 관련 수행사 인력 분등른 6월 30일까지 모든 권한(방화벽, DRM, NAC, 메신저, 메일 등..)이 들어가 있다.
   안정화기간 한달을 가지고 해당 결과로 나오는 정관장 멤버스 회원BO, FO 운영 담당 예정입니다.

느낀점 : 해당 프로젝트를 진행하면서, 많은 회의들을 했고 어떤 과정에서 무엇을 챙겨야하고 필요하며 이런식으로 프로젝트가 진행되는구나를 알게 된 큰 경험이 되었습니다.
단위테스트는 모듈, 기능별로 하나씩 눌러보면서 협의된 부분들이 실행되는 것을 경험했고, 꼼꼼히 살펴야되는 부분이 많았던 것 같습니다.
더불어 방화벽관련하여서 많은 고생을 했던 것 같습니다. 결재테스트를 하거나 특정 테스트를 위해 필요한 방화벽이 엄청 많았고 어느 사유로 안되는지 명확히 현황파악이 안되어 이슈화되는 
부분들을 경험하면서, 일정을 챙기는데 있어서 고려해야되는 부분들을 알게 되었습니다.
이러한 지식과 경험을 가지고 다음 어떤 프로젝트를 하게 될때는 보다 더 나은, 한단계 성장한 사람이 되었을 것이라고 생각합니다.

총 3차로 3년 예정 -> 총합 330억에서 300억 비용 예산 산정이나, 2차/3차 미정

* IT 프로젝트 전과정 (8단계로 구분)
   1. 프로젝트 제안 및 수주
       - 고객사 사전 분석 (RFI) : 고객사가 프로젝트 발주를 위해 사전 정보 수집하는 단계 
       - 사업공고 및 제안요청 (RFP) : 고객사의 RFP 기반의 제안요청
       - 제안 및 견적 (RFQ)  : 고객사 사업 공고에 따른 제안 및 견적 작업, 제안 발표
       - 낙찰(계약) : 업체 선정 및 계약
   2. 프로젝트 계획 및 시작
       - 팀구성 : 내부 및 외주 포함 팀 구성
       - 프로젝트 진행 전반 계획 : 현업 TFT와 진행 계획 
       - 기본 진행 일정 계획 : 주요 업무 단위별 러프한 일정 계획 
       - 프로젝트 본격 시작 : 착수보고 (관련자 사업보고, 업무요청, 진행 방향등 공유)
   3. 기획
       - 요구사항 분석
       - 개발 기능 정의 (이후 WBS 상세 일정)

       - 전체 화면 디자인 작업 
   5. 퍼블리싱
       - 일정에 따른 퍼블리싱
       - 퍼블리싱 보완 및 지원
   6. 개발
       - (1-5과정중)서버-네트워크 환경 구성
       - (1-5과정중)개발 환경 구성  
   7. 테스트
       - 테스트 케이스 작성
       - 테스트 시나리오 작성
       - 단위테스트 및 보완
       - 통합테스트 및 보완
   8. 오픈 안정화
       - 사전교육 (사용자, 관리자 메뉴얼 기반)
       - 오픈준비 (데이터전환 계획, 배포 계획등 이행계획 수립)
       - 오픈
